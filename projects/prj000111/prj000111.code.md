## Core Classes

### APIDocGenerator
**File**: `api_docs_generator.py`, lines 12-142

**Responsibility**: Main API documentation generator class
- Registers API endpoints with metadata
- Extracts parameter information from function signatures
- Generates OpenAPI 3.0 specifications
- Creates markdown documentation
- Exports specifications as JSON

**Key Methods**:
1. `__init__()`: Initialize endpoints list
2. `register_endpoint(func, method, path, description)`: Register a function as an API endpoint
3. `_extract_parameters(sig, type_hints)`: Extract parameters from function signature
4. `_extract_return_type(type_hints)`: Extract return type information
5. `generate_openapi_spec(title, version)`: Generate OpenAPI 3.0 spec
6. `_openapi_parameters(parameters)`: Convert to OpenAPI parameter format
7. `generate_markdown(title)`: Generate markdown documentation
8. `export_json(spec)`: Export specification as JSON string

**Dependencies**:
- `typing`: For type hint extraction
- `json`: For JSON operations
- `inspect`: For function signature parsing

## Usage Example

```python
from api_docs_generator import APIDocGenerator

# Create generator
gen = APIDocGenerator()

# Register endpoints
def get_user(user_id: int) -> dict:
    """Get user by ID."""
    return {"id": user_id}

gen.register_endpoint(get_user, method="GET", path="/users/{id}")

# Generate specifications
spec = gen.generate_openapi_spec(title="My API", version="1.0.0")
markdown = gen.generate_markdown()
json_output = gen.export_json(spec)
```
