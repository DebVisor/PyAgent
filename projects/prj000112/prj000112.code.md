## Data Structures

### RuffViolation (Dataclass)
**File**: `ruff_precommit.py`, lines 14-22

**Responsibility**: Represents a single Ruff linting violation
**Fields**:
- `file: str` - Path to file with violation
- `line: int` - Line number of violation
- `column: int` - Column number of violation
- `code: str` - Violation code (e.g., "E501")
- `message: str` - Human-readable error message
- `severity: str` - Severity level (default: "warning")

**Usage**:
```python
violation = RuffViolation(
    file="app.py",
    line=10,
    column=80,
    code="E501",
    message="Line too long"
)
```

## Core Classes

### RuffConfig
**File**: `ruff_precommit.py`, lines 25-77

**Responsibility**: Manages Ruff configuration loading and validation
**Key Methods**:
1. `__init__(config_path)`: Initialize with config file path
2. `_load_config()`: Load configuration from TOML file or use defaults
3. `validate()`: Validate configuration structure and values
4. `get_config()`: Retrieve current configuration

**Configuration Keys**:
- `line-length`: Maximum line length (default: 100)
- `select`: Rules to enable (default: ["E", "F", "W"])
- `ignore`: Rules to ignore (default: ["E203", "W503"])
- `exclude`: Directories to exclude
- `per-file-ignores`: Per-file rule overrides

**Implementation Details**:
- Simple TOML parsing (basic key=value extraction)
- Graceful fallback to defaults if file missing
- Type conversion for integer values
- Validation of required fields

### RuffVersionManager
**File**: `ruff_precommit.py`, lines 80-129

**Responsibility**: Manages Ruff version detection and compatibility checking
**Key Methods**:
1. `__init__(min_version)`: Set minimum version requirement
2. `get_installed_version()`: Detect installed Ruff version via subprocess
3. `compare_versions(v1, v2)`: Compare version strings (-1, 0, 1)
4. `check_version_compatibility()`: Verify installed meets minimum

**Version Comparison Logic**:
- Parses semantic versioning (X.Y.Z)
- Compares each component numerically
- Returns -1 (v1 < v2), 0 (equal), 1 (v1 > v2)
- Handles variable-length version strings

**Output**:
```python
(True, "Ruff 0.5.0 OK")  # Compatible
(False, "Ruff 0.1.0 < required 0.5.0")  # Not compatible
(False, "Ruff not installed")  # Not installed
```

### PreCommitRuffHook
**File**: `ruff_precommit.py`, lines 132-186

**Responsibility**: Executes Ruff checks and collects violations
**Key Methods**:
1. `__init__(config)`: Initialize with RuffConfig
2. `run_check(files)`: Execute Ruff on specified files
3. `format_violations()`: Format violations for display

**Execution Flow**:
1. Execute `ruff check --json` command
2. Parse JSON output from Ruff
3. Convert to RuffViolation objects
4. Return success status and violations list

**Error Handling**:
- Catches FileNotFoundError (Ruff not installed)
- Catches subprocess.TimeoutExpired (timeout)
- Gracefully handles JSON parsing errors
- Returns empty violation list on error

**JSON Output Parsing**:
```json
[
  {
    "filename": "app.py",
    "line_number": 10,
    "column_number": 80,
    "code": "E501",
    "message": "Line too long"
  }
]
```

## Usage Example

```python
from ruff_precommit import RuffConfig, RuffVersionManager, PreCommitRuffHook

# Check configuration
config = RuffConfig("pyproject.toml")
is_valid, errors = config.validate()

# Check version
version_mgr = RuffVersionManager(min_version="0.1.0")
compatible, status = version_mgr.check_version_compatibility()
print(status)

# Run checks
if compatible:
    hook = PreCommitRuffHook(config)
    success, violations = hook.run_check(["src/"])
    if not success:
        print(hook.format_violations())
```
