# Design

This file defines the role of design documentation and where it should live.

## Purpose
- Clarify what belongs in design artifacts versus implementation or planning artifacts.

## Where to write design docs
- Design should be documented in `docs/project/prjXXX-*/plan.md` and `docs/project/prjXXX-*/brainstorm.md`.
- Use `docs/architecture/archive/` for deep dive reference materials when a design decision is already captured there.
