# prj0000075 — ci-simplification

## Goal
The GitHub Actions CI directory has grown to 5 workflow files despite the user's intent to keep it minimal and rely on local testing via pre-commit hooks. This project:

1. **Audits** all 5 `.github/workflows/*.yml` files: `ci.yml`, `core-quality.yml`, `pm.yml`, `quality.yml`, `testing-infra.yml`
2. **Evaluates** whether each workflow adds value over what pre-commit hooks already do (or can do)
3. **Decides** (via @2think → @3design) whether to remove workflows entirely, keep one minimal CI file, or merge them
4. **Implements** the decision: removes or consolidates workflow YAML, ensures pre-commit covers the same checks (ruff, mypy, flake8, pytest smoke)
5. **Documents** the rationale so future contributors know the deliberate choice

## Scope boundary
- `.github/workflows/*.yml` — audit and modify/remove
- `.pre-commit-config.yaml` — expand if needed to cover gaps left by removed workflows
- `docs/` — update CI guidance in `docs/setup.md` or equivalent
- Do NOT touch any Python/Rust/TS source code or test files

## Branch Plan

## Acceptance Criteria
1. Each workflow file has been audited and a documented decision (keep / merge / remove) exists in `.think.md`
2. Post-change: `.pre-commit-config.yaml` covers every check previously in a removed workflow
3. At most ONE GitHub Actions workflow file remains (or zero if pre-commit fully covers everything)
4. `docs/setup.md` (or equivalent) updated to document local-first testing instructions
5. All pre-commit hooks pass locally
6. A `tests/structure/` or `tests/ci/` test validates that no unexpected workflow files exist
